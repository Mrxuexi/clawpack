# ClawPack

ClawPack 是一个面向 OpenClaw 类 Agent 的应用层打包与分发工具。

它解决的问题很简单：把一个已经调好的 OpenClaw Agent，标准化地导出、导入和验证。它不是容器，也不是虚拟机，而是建立在这些基础设施之上的应用层分发包。

## Why

OpenClaw 的可用状态不只是代码或配置，还包括：

* workspace
* prompts / skills
* provider 配置
* channel 配置
* state / memory
* 部分运行态信息

这些内容可以被打包成一个更适合迁移和复用的 Agent 包。Docker 能分发运行环境，但不能直接表达“哪些内容是模板、哪些内容是状态、哪些内容需要重绑”。ClawPack 的目标就是补上这一层。

## Goals

ClawPack v0.1 只做一个最小闭环：

* 识别一个 OpenClaw 实例
* 导出为标准包
* 导入到新环境
* 验证导入结果

## Non-goals

当前版本不做：

* Web 平台
* 在线 Registry
* 模板市场
* 多租户托管
* 复杂加密
* 高级脱敏
* 跨所有版本兼容保证

## Package Types

ClawPack v0.1 定义两类包。

### Template Pack

用于分享和复用。

特点：

* 包含 workspace 和非敏感配置
* 默认不包含完整 state
* 默认不包含敏感凭证
* 适合复制能力，不适合复制身份

### Instance Pack

用于完整迁移。

特点：

* 可包含 config、workspace、state
* 可能包含运行态信息
* 仅适合受信任环境导入

## CLI

v0.1 只包含 4 个命令：

* `clawpack inspect`
* `clawpack export`
* `clawpack import`
* `clawpack verify`

### inspect

识别当前 OpenClaw 实例的基本结构和风险，给出推荐导出类型。

### export

将实例导出为标准 ClawPack 包，支持：

* `template`
* `instance`

### import

将 ClawPack 包导入目标环境。第一版只支持最简单的 restore 模式。

### verify

检查导入后的实例是否结构完整、基础可用。

## Package Format

v0.1 采用简单的标准包结构，至少包含：

* `manifest.json`
* `config/`
* `workspace/`
* `state/`（仅 instance）
* `reports/`

建议文件扩展名：

* `.clawpack`

## Manifest

每个 ClawPack 包都必须包含 `manifest.json`，至少描述：

* schema version
* pack type
* pack id
* created time
* source product/version
* included paths
* sensitive flags
* risk level

## Risk Levels

v0.1 只定义三档风险级别：

* `safe-share`
* `internal-only`
* `trusted-migration-only`

通常：

* template 更接近 `safe-share`
* instance 默认至少是 `trusted-migration-only`

## Output

所有命令都应支持两种输出格式：

* `text`
* `json`

其中：

* `text` 面向人类使用
* `json` 面向自动化和未来 Web 调用

## Success Criteria

ClawPack v0.1 的最小成功标准是：

1. 用户可以对一个 OpenClaw 实例执行 `clawpack inspect`
2. 用户可以导出一个 template 或 instance 包
3. 用户可以在新目录执行 `clawpack import`
4. 用户可以执行 `clawpack verify`
5. 系统能给出明确的成功、警告或失败结果

## Scope Summary

ClawPack v0.1 是一个最小的 OpenClaw Agent 打包与分发 CLI，先打通：

**inspect → export → import → verify**

后续再逐步扩展到：

* 加密
* 更强脱敏
* merge 导入
* 本地 registry
* Web 平台
